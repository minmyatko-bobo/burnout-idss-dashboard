# Explainable Burnout IDSS — Streamlit dashboard (deployment package)

A **self-contained, deploy-ready** copy of the IDSS presentation layer. It reads a pre-generated, fully synthetic workforce (`data/sehim_workforce.json`) — so it needs **no models, no raw data, and no heavy ML libraries** at runtime, which makes it fast and safe to host publicly.

Pages: Executive Summary · Department Heatmap · Employee Risk Cards (with PDF export) · Intervention Recommendations. Sidebar role selector enforces RBAC (HR_Admin = all; HR_Manager = own department).

## Run locally
```bash
pip install -r requirements.txt
streamlit run dashboard.py
```

## Deploy to Streamlit Community Cloud (free)

> You'll do these steps in your own GitHub and Streamlit accounts. This folder is everything the deployed app needs — do **not** add the raw `Dataset/` or `References/` folders (the `.gitignore` already blocks them; the EWCS data is UK Data Service-licensed and must not be made public).

**1 — Put this folder in a public GitHub repo.**
Easiest via GitHub web:
- Go to https://github.com/new → name it e.g. `burnout-idss-dashboard` → Public → Create.
- On the repo page: **Add file → Upload files** → drag in everything inside `streamlit_app/` (the files, not the folder): `dashboard.py`, `dashboard_logic.py`, `requirements.txt`, `README.md`, the `data/` folder, and the `.streamlit/` folder → **Commit changes**.

Or via command line (from inside the `streamlit_app/` folder):
```bash
git init && git add . && git commit -m "Burnout IDSS dashboard"
git branch -M main
git remote add origin https://github.com/<your-username>/burnout-idss-dashboard.git
git push -u origin main
```

**2 — Deploy.**
- Go to https://share.streamlit.io and sign in with GitHub (authorise it once).
- Click **Create app → Deploy a public app from GitHub**.
- Repository: `…/burnout-idss-dashboard` · Branch: `main` · **Main file path: `dashboard.py`**.
- (Optional) **Advanced settings → Python version 3.11.**
- Click **Deploy**. First build takes ~2–3 minutes while it installs `requirements.txt`.

**3 — Get your URL.**
You'll receive a public link like `https://<your-app>.streamlit.app`. Paste it into:
- the expert-invitation email template, and
- the scenario packet / Google Form.

## Updating the demo data
The committed workforce is generated by `app/sehim_api.py` in the main project (`python app/sehim_api.py` → `data/sehim_workforce.json`). To refresh the deployed demo, copy the regenerated `sehim_workforce.json` into `streamlit_app/data/` and push again — Streamlit Cloud redeploys automatically on each push.

## Notes / limits
- All displayed data is **synthetic** — no real employees, safe to host publicly.
- Streamlit Community Cloud free tier (~1 GB RAM) is ample; this app loads a small JSON, not the ML models.
- Apps on the free tier sleep after inactivity and wake on the next visit (a few seconds) — fine for an evaluation window.
